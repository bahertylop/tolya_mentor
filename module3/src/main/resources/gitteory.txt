ветки - указатели на коммиты

git merge объединяет изменения на ветках в общий коммит

Rebase (перебазирование) — один из способов в git, позволяющий объединить изменения двух веток.
У этого способа есть преимущество перед merge (слияние) — он позволяет переписать историю ветки,
придав истории тот вид, который нам нужен.

Из документации — это наложение коммитов поверх другого базового коммита. Под базовым понимается
тот коммит, к которому применяются коммиты выбранной ветки.

после rebase смердить ветки и сделать коммит слияния, чтобы было понятно, когда были объединены ветки,
после можно удалить зависимую ветку


перемещение между коммитами:
    ^       перемещение на один коммит назад
    ~<num>  перемещение на несколько коммитов назад

    использование относительных ссылок, main^ или main^^ или main~<2>

    также при помощи HEAD^

    git branch -f main HEAD~3 - привязывает ветку к другому коммиту



отмена изменений:
    git reset - для локальных репов
    git revert - для удаленных, если хотим поделиться изменениями (делает коммит с изменениями обратными коммиту)
